Git Topic



Definisi

Git adalah sistem kontrol versi yang digunakan oleh pengembang perangkat lunak untuk melacak perubahan dalam kode sumber proyek. Ini memungkinkan pengembang bekerja secara bersamaan pada proyek yang sama. Dengan Git, pengembang dapat membuat cabang (branch) untuk memodifikasi kode tanpa mempengaruhi kode utama (master), dan kemudian menggabungkan perubahan tersebut (merge) kembali ke kode utama setelah selesai.

Git Bash adalah terminal atau antarmuka baris perintah yang memungkin pengguna untuk menjalankan perintah Git pada sistem operasi Windows.

GitHub adalah platform berbasis web yang memanfaatkan Git sebagai sistem kontrol versi (VCS). Ini berfungsi sebagai layanan hosting untuk repositori Git, memungkinkan pengembang untuk menyimpan dan berbagi proyek perangkat lunak secara online.


Basic Git Commands


Untuk menggunakan Git, para pengembang menggunakan perintah-perintah khusus untuk menyalin, membuah, mengubah, dan menggabungkan kode. Berikut adalah beberapa perintah umum untuk menggunakan Git :

  • git init: perintah ini digunakan untuk memulai repositori git baru di dalam suatu folder.
  • git status: perintah ini digunakan untuk memberikan gambaran tentang status perubahan proyek.
  • git log: perintah ini digunakan untuk melihat catatan perubahan atau riwayat (change log) pada repositori git.
  • git add: perintah ini digunakan untuk menambahkan perubahan pada file ke daftar yang akan di-commit, siap untuk dimasukkan ke dalam riwayat proyek.
  • git commit: perintah ini digunakan untuk menyimpan snapshot perubahan yang sudah di-"add" ke dalam riwayat proyek dengan pesan yang menjelaskan perubahan yang dilakukan.
  • git push: Perintah ini digunakan untuk mengirimkan perubahan yang di lakukan secara lokal ke repositori online.
  • git branch: Perintah ini digunakan untuk menampilkan daftar cabang yang sedang dikerjakan secara lokal.
  • git fetch: perintah ini digunakan untuk mengambil informasi terbaru dari repositori online ke komputer lokal tanpa mengubah kode yang sedang dikerjakan.
  • git checkout: perintah ini digunakan untuk beralih antara cabang-cabang yang ada di repositori Git. Ini juga dapat digunakan untuk menciptakan cabang baru dan berpindah ke cabang tersebut.
  • git clone: perintah ini digunakan untuk membuat salinan lengkap proyek dari repositori online ke komputer lokal, termasuk riwayat perubahan dan cabang.
  • git pull: perintah ini digunakan untuk mengambil perubahan terbaru dari repositori online dan menggabungkannya dengan kode lokal.
